## シグナル検出（calc_atr / detect_signals）

- RSI 過売り反発 / 過買い警戒
- MACD ゴールデン / デッドクロス
- 移動平均 MA25 vs MA75 ゴールデン / デッドクロス
- ボリンジャー ±2σ タッチ・突破（🆕 2026-05-28: bb_pos 0.3/0.7 必須化でバグ修正）
- 直近 20 本 高値・安値ブレイク
- 🆕 **2026-05-28: チャートパターン P1**（自前ロジック、scipy 不要）
  - **ダブルトップ** (`double_top`): 反転型、2 高値の差 0.5% 以内 + ネックライン -2% 割れ + 現close 下抜け
  - **ダブルボトム** (`double_bottom`): 反転型、2 安値の差 0.5% 以内 + ネックライン +2% 突破 + 現close 上抜け
  - **三角持ち合い** (`triangle_squeeze`): 中立型、高値切下げ + 安値切上げ + BB 幅縮小（直近平均の 70% 以下）

### 反転型パターンの特別扱い
ダブルトップ / ダブルボトム は **モメンタムフィルタをバイパス**。
理由: 反転時点では entry_vs_ma25 がまだ転換前方向（モメンタムと逆）。
新案F の閾値で弾くと反転シグナルが永遠に通らない。
代わりに 2 つのピーク/谷の近接精度（tolerance_pct）とネックラインブレイク幅（neckline_drop/rise_pct）で品質保証。

## 環境警戒システム（取引推奨度 A〜D）

シグナル発火時に以下を統合判定:

- 📅 **重要指標近接**（FOMC / 雇用統計 / CPI / 日銀等、`economic-events.json` 参照）
- 📊 **VIX レベル**（25 超で警告、30 超で禁止、24h +20% で警告）
- ⚡ **ATR レジーム**（過去 30 日平均との倍率、3x 超で取引禁止）
- 🚨 **危機キーワード**（戦争・介入・暴落等のニュース検知）
- 🏖️ **市場休場日近接**（当日休場なら env_score を 1 段階引き下げ）

## 🆕 2026-05-28 実装: モメンタムフィルタ + 市況判定 + バグ修正

### Step A: モメンタムフィルタ（新案F）

ユーザーの観察「勝てる銘柄は人気・モメンタムが出ている銘柄」を動的判定で実装。**銘柄ブラックリストは採用せず**、モメンタムが出ている銘柄を動的に拾う。

**フィルタ式**:
```
dsign = +1 (long) / -1 (short)
ma_ratio = (ma25 - ma75) / ma75 × 100
entry_vs_ma25 = (entry - ma25) / ma25 × 100
range_pos = (entry - recent_low) / (recent_high - recent_low)
momentum_score = (ma_ratio + entry_vs_ma25 + (range_pos - 0.5)) × dsign

メール送信条件:
  entry_vs_ma25 × dsign > 0.5  OR  momentum_score >= 1.0
```

**バックテスト結果（N=139）**:
- 通過: 29 件、勝率 **55.2%**
- 拒否: 110 件（SL 回避 81、機会損失 29 = TP 取り逃し）
- NKD=F 8/9 = 88.9%、CL=F 2/4 = 50%、NQ=F 3/4 = 75%、BTC 2/3 = 67%
- GC=F は完全に弾かれた（動的判定でブラックリスト相当の効果）

**重要**: メール送信されないシグナルも `signals-log.json` には記録（`momentum_filter` フィールド）。次回検証で「拒否したシグナルが実際どれだけ負けていたか」を測定可能。

### Step B-1: `bb_lower_touch` バグ修正

検証で発覚: `bb_lower_touch` 全 43 件のうち **15 件 (35%)** が実際は BB バンドの中央〜上半分（bb_pos 0.5-0.8）で発火していた。勝率 13.3% で構造的に負け。

**修正**: `bb_pos < 0.3` 必須条件を `detect_signals()` に追加。`bb_upper_break` も対称的に `bb_pos > 0.7` 必須に。

### Step B-2: `ma_golden` 単独シグナルの送信拒否

N=11 で 0 勝（全敗）。MA はラグ指標で、ゴールデンクロス時点ではトレンド初動が既に終わっている可能性。

**動作**: `signal_count == 1 and signal_types == ['ma_golden']` のとき、メール送信スキップ（複合発火時は他シグナル主導なので通す）。

### Step C: ADX + Choppiness Index の記録

`indicators_at_signal` に以下を追加（フィルタには使わず記録のみ、次回検証用）:
- `adx` （Average Directional Index、≥25 でトレンド、<20 でレンジ）
- `chop` （Choppiness Index、>61.8 でレンジ、<38.2 でトレンド）
- `ma_dev_pct` （|ma25-ma75| / ma75 × 100）
- `ma_dir` （up / down）
- `regime` （range / trend_up / trend_down）

3 ヶ月後の再評価（2026-08-28）で、本物の ADX/Choppiness と擬似指標（MA 乖離率）の精度差を測定して、フィルタロジック自体を ADX ベースに切り替えるか判断。

### MA 丸め桁数の改善

FX で `ma25 == ma75` になる丸め問題（検証で 21/139 件が指標欠損扱い）を回避するため、`indicators_at_signal` の MA / BB / recent_high/low の丸め桁数を 2 桁 → 4 桁に拡張。
