## ⚠️ 注意点・既知の問題

### 重要な運用ルール
1. **HTML 6 コアファイルを SYNC_FILES に絶対入れない**（過去日付巻き戻り事故あり）
2. **新記事追加は 8 ステップルール厳守**（更新履歴は常に 5 件キープ）
3. **速報系記事を書く前に「日付の事実確認」を WebSearch で行う**
4. **ローカルの signals-log.json は触らない**（workflow が GitHub 側で管理）

### Gemini モデルチェーン
レート制限対策で `gemini-2.0-flash-lite` → `gemini-2.5-flash-lite` → `gemini-2.0-flash` → `gemini-2.5-flash` の順でフォールバック。

### yfinance の落とし穴
- ローカルからの連続アクセスでレート制限される
- GitHub Actions 側は問題なく動作
- `NKD=F` は関連ニュースが取れないことが多い（先物のため）

### 環境イベントカレンダー
`economic-events.json` は手動メンテ。月初に当月分を追加すること。
