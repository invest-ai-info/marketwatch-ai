**作成日**: 2026-05-29

### 根本原因
現エンジンは **テクニカル指標のルール一致だけで発火**し、方向もテクニカル任せ。
- ファンダ/マクロは方向を決めず、`check_environment` の A〜D（リスクオフのブレーキ）として後付けされるだけ。
- 勝敗ログ（signals-log）が発火判断にフィードバックされていない（ループが開いている）。

## 2. 設計確定事項（ユーザー決定）

1. **順序を逆転**：ファンダ/マクロで方向を決めてから、テクニカルでタイミングを測る（トップダウン）。
2. **リスクオン/オフ層を最上段に追加**。
   - リスクオン → 強気（強いリスク資産をロング）
   - リスクオフ → 「弱い方に強気」＝**🅐 弱いリスク資産をショート／安全資産をロング**（順張り・相対強弱）。
3. **逆張りの飛びつき禁止**：売られすぎ/買われすぎへの即時エントリーは「どこまで行くか分からない」ため絶対にやらない。
4. **「落ち着いてから入る」**：逆張りシグナルは監視フラグに格下げし、**🅰奪回 ＋ 🅱底固め ＋ 🅲ボラ収縮 の3点全成立**を待って初めて発火（最も保守的な確認方式）。

## 3. アーキテクチャ（4階建て）

```
Layer 0 ┃ リスク環境判定         リスクオン / 中立 / リスクオフ
        ▼
Layer 1 ┃ 方向バイアス（順張り・相対強弱 🅐）   資産ごとに 強気/弱気/中立
        ▼
Layer 2 ┃ テクニカル＝タイミングのみ           3点確認を満たしたら発火
        ▼
Layer 3 ┃ リスクブレーキ（既存）               環境スコアA〜D + 往復ビンタ → 見送り
```

### Layer 0：リスク環境判定（合成スコア）
既存データで算出。各入力をスコア化して合算（閾値は Phase 2 で較正）。

| 入力 | リスクオン寄り(+) | リスクオフ寄り(−) |
|---|---|---|
| VIX 水準 | <15: +2 / 15–20: 0 | 20–25: −1 / >25: −2 |
| VIX 5日方向 | 低下: +1 | +20%以上急騰: −1 |
| 通貨強弱（既存マトリクス） | 豪ドルが強い: +1 | 円・ドルが強い: −1 |
| 金 vs 株（5日リターン差） | 株が金を上回る: +1 | 金が株を大きく上回る: −1 |
| 危機キーワード（既存） | — | 1件: −1 / 3件以上: −2 |

合算 → **RISK_ON（≥+2）/ NEUTRAL / RISK_OFF（≤−2）**

### Layer 1：方向バイアス（相対強弱・順張り）
資産を「リスク資産」「安全資産」に分類：
- リスク資産: 株価指数(ES/NQ/YM/NKD/FTSE)、BTC、原油、豪ドル系
- 安全資産: 金(GC)、円、ドル

| リスク環境 | リスク資産 | 安全資産 |
|---|---|---|
| RISK_ON | **ロングのみ許可**（強いものを買う、売り禁止） | 中立〜弱気 |
| RISK_OFF | **ショートのみ許可**（弱いものを売る） | **ロング許可**（金・円・ドル買い） |
| NEUTRAL | 上位足トレンド方向のみ許可 | 同左 |

FX は通貨強弱マトリクスで「強い通貨ロング／弱い通貨ショート」を基本とし、リスク環境で安全通貨（円・ドル）寄せを加味。

### Layer 2：テクニカル・タイミング（「落ち着いてから」確認）
逆張りシグナル（rsi_oversold_bounce, bb_lower_touch, rsi_overbought 等）は**即発火せず監視フラグ化**。以下3点が**全部**揃い、かつ Layer 0/1 と方向一致したら発火：

- **🅰 奪回（reclaim）**: 終値が直前足の高値を上抜け OR 短期MA(5〜10)の上に回復（売りは鏡像）
- **🅱 底固め**: 直近3本で新安値を更新しない かつ 安値切り上げ（higher low）（売りは鏡像）
- **🅲 ボラ収縮**: 直近足の値幅／BB幅が直近平均より縮小、ATR が落ち着く

順張りシグナル（high_break, macd_golden, ma_golden 等）は Layer 0/1 と方向一致が条件。`detect_first_pullback`（トレンド中の押し目待ち）を主役に昇格候補。

**トレードオフ（合意済）**: 反発初動は取り逃すが、勝率が上がりナイフ掴みの事故を消す。安全最優先。

### Layer 3：リスクブレーキ（既存維持）
`check_environment` の A〜D、往復ビンタ防止（12h反転検知）、銘柄別クールダウン。環境Dは無条件見送り。

### 確信度（⭐）の扱い
現状の勘ベース重みは破棄。Layer 0〜2 を通過した実績で**勝率較正し直す**まで、誤誘導を防ぐため⭐表示は控えめ運用 or 一時非表示を検討。

## 8. L2 ファンダ・ブリーフィング（2026-05-29 実装状況）

「ファンダ＝最重要、まずニュースの信頼性」というユーザー判断に基づき、Layer0/1 の“厚い”入力を **Claude の多サブエージェント・ブリーフィング**で生成する方式（L2）を採用。

### 仕組み
```
[定期] ブリーフィング（多サブエージェント）  収集→裏取り→合成
   ├ 収集: 4レンズ並列（中銀・公式/通信社/日本語・国内/地政学）で WebSearch
   ├ 裏取り: ファクトチェッカーが重複統合＋クロス照合、信頼度(HIGH/MID/LOW)採点、伝聞は除外
   └ 合成: ストラテジストが信頼度加重で risk_regime ＋ 資産別 bias を生成
        ↓ 成果物
   fundamental-context.json （risk_regime / assets[].bias / top_news(信頼度・裏取り付き) / discarded）
        ↓ cron エンジンが読むだけ
[4hごと] generate_technical_alerts.py が Layer0/1 入力として参照
```

### 信頼度フレーム
- ソース格付け: tier1 一次/公式（中銀・統計・IR）＞ tier2 通信社/金融専門 ＞ tier3 一般大手 ＞ tier4 アグリゲーター/SNS
- クロス照合: 独立した tier1-2 が2社以上→HIGH。単一 tier1→HIGH、単一 tier2→MID、単一 tier3-4/伝聞→LOW or 除外
- 重要度(materiality)・鮮度で加重。LOW に引っ張られないこと

### 試走結果（2026-05-29、5資産）
- 6エージェント / 約238kトークン / 6分。収集32→採用11（21件は重複・低信頼で排除）
- 出力例: risk_regime=NEUTRAL(MID)、日経BULLISH(MID)・S&P BULLISH(MID)・ドル円/金 NEUTRAL・BTC BEARISH(LOW)
- 品質OK: BEA一次統計=tier1/HIGH、Reuters/Bloomberg=tier2、corroborated_by 列挙、BTCは薄い材料で conviction=LOW と自己抑制、除外理由も記録。日付もプロジェクト世界(2026-05-29)と整合（WebSearch で成立、L1課金ソース不要）

### エンジン連携（実装済・記録のみ）
- `load_fundamental_context()` / `briefing_bias_for()` を追加（commit `058c4d9`）
- 各シグナルに `fundamental_context`（source/regime/asset_bias/conviction/**bias_aligned**）を記録
- 不在・鮮度切れ（>30h）は計算 regime に安全フォールバック。FX は `=X` 正規化で照合
- **発火・メールは未変更（Phase 1 と同じ記録のみ）**。Phase2 で「bias_aligned=False を弾いていたら勝率は?」を検証

## 9. Phase 2 検証結果（2026-06-09）＝ Layer 0/1 フィルタ不採用

signals-log **642件／決済534件**（過去最大N）で「Phase 1 で記録した Layer 0/1 フィールドでフィルタしていたら勝率は？」を反実仮想検証した（ツール: `research/phase2_backtest.py`、レポート: `research/phase2_report.md`。`research/` は SYNC 対象外の研究用フォルダ）。

### 判定（§7 成功指標に照らして）
**不合格 → Phase 3 への昇格は中止。** ただし Phase 1 の記録は継続し、別レジームのデータが貯まったら再評価する。

| 検証項目 | 結果 | 解釈 |
|---|---|---|
| Layer1 一致（bias=方向） | **11.8%**（n17, CI 3–34%） | 一致して入ると**むしろ大負け** |
| Layer1 不一致 | **56.6%**（n53, CI 43–69%）✅ | CI 非重複の明確な逆転 |
| ファンダ一致（fc.aligned=T） | 34.8%（n46） | 6/6 分析（True 17% vs なし45%）を大Nで再確認 |
| regime=RISK_ON | 72.7%（n11）✅ | 有望だが極小N＝監視扱い |
| ショート全体 | 50.4%（n115, +0.18R） | **旧分析の24.1%から反転**＝非定常性の生きた証拠 |
| ma_golden | 23.3%（n30, CI上限41%<43%） | 統計的に有意な死に筋（§4 維持） |
| macd_dead | **49.4%（n81）** | §4 の「死に筋」判定は小サンプル（n25）の誤り → **除外リストから外す** |

### なぜ設計が棄却されたか（構造的理由）
このエンジンの実エッジは「**売られすぎ＝パニックの逆張り**」（6/3確定）。トップダウンの順張りゲート（Layer 0/1）は、その勝ち筋（バイアスに逆らう逆張り）を弾き、負け筋（バイアス方向への飛びつき）を通す——**エッジとフィルタの向きが構造的に逆**だった。これは 6/6 の「fib_pullback 停止＝シグナルを100%テクニカルに戻す」判断とも整合する。

### 禁止事項（過適合ガード）
- **「不一致を逆に張る」フィルタ反転は禁止**。n53 の単一レジーム逆転を反転採用するのは過適合の典型。記録継続→前向き再現（n≥40・別レジーム）まで動かさない。

### 残バグ（軽微・既知）
- エンジンは `outcome='tp2'` を出さない（勝ち215件中52件=24%が `hit_tp2_at` でTP2到達、`sl`×`hit_tp2_at` 矛盾レコード1件）。分析側は `hit_tp2_at` 参照で対応済み。エンジン側修正は急がない（表示・記録のみの問題）。
